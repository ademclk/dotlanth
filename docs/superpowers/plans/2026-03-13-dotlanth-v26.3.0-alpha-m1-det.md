# Dotlanth v26.3.0-alpha M1 DET Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add strict deterministic mode plumbing, persist the selected mode in inspectable execution metadata, and fail closed before unsupported strict-mode side effects execute.

**Architecture:** Add a shared `DeterminismMode` type at the runtime boundary, thread it from `dot run` through runtime execution and bundle metadata, and enforce an initial strict-mode denial path at the VM-to-host boundary. Keep the first milestone intentionally narrow: mode selection, metadata visibility, and fail-closed semantics only.

**Tech Stack:** Rust workspace, `clap`, `serde_json`, existing `dot_vm`/`dot_ops`/`dot_rt`/`dot_artifacts` crates, `cargo test`

---
