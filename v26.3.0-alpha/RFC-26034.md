# RFC-26034: Deterministic CLI UX Spec

## Status
Draft (target for v26.3.0-alpha)

## Summary
Defines the CLI UX additions for deterministic mode and replay validation in v26.3.

## Commands

### `dot run --deterministic=strict <entry.dot>`
Behavior:
- Runs the entrypoint under strict deterministic policy.
- Prints run id and selected determinism mode.
- On denial, prints a deterministic policy failure with actionable reason.

### `dot inspect <run_id>`
Additional behavior for v26.3:
- Prints determinism mode.
- Prints validator eligibility.
- Prints violation summary and budget summary.

### `dot validate-replay <run_id>`
Behavior:
- Replays the run using the supported replay contract.
- Prints one of: `valid`, `invalid`, `unsupported`.
- Includes concise mismatch details when invalid.

## Exit Codes
- `0` on success / valid result
- non-zero on invalid result, unsupported state that should fail automation, or tooling/runtime failure

## UX Principles
- Keep output copy/pasteable.
- Favor concise summaries over verbose logs.
- Distinguish clearly between:
  - capability denial
  - deterministic policy denial
  - replay proof mismatch
