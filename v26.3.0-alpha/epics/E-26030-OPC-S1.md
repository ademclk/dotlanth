# Story E-26030-OPC-S1: classification schema and metadata wiring

## Epic
- E-26030-OPC

## Story
As a tooling author, I want opcode determinism metadata, so policy, artifacts, and validators use the same categories.

## Acceptance Criteria
- Metadata model supports `pure`, `controlled_side_effect`, and `non_deterministic`.
- In-scope opcodes can expose their category and required capability context.
- Classification is stored centrally enough to avoid policy drift.

## Tasks (0.5-2 days)
- E-26030-OPC-S1-T1 (1.0d): Implement the classification enum/metadata shape in runtime or VM code.
- E-26030-OPC-S1-T2 (1.0d): Attach metadata to the in-scope opcode registry and expose it to enforcement/tooling.
