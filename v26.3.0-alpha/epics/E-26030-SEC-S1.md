# Story E-26030-SEC-S1: boundary enforcement uses classification and mode

## Epic
- E-26030-SEC

## Story
As a security reviewer, I want deterministic policy enforced at the opcode/host boundary, so prohibited operations are denied before they happen.

## Acceptance Criteria
- Boundary checks consult determinism mode and opcode classification.
- `non_deterministic` operations are denied in strict mode before host execution.
- Controlled side effects still require declared capability approval.

## Tasks (0.5-2 days)
- E-26030-SEC-S1-T1 (1.0d): Compose determinism and capability checks at the opcode/host boundary.
- E-26030-SEC-S1-T2 (1.0d): Add denial-path tests proving host execution does not occur after strict-mode rejection.
