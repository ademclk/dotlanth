# Story E-26030-DET-S2: strict mode rule enforcement and failure semantics

## Epic
- E-26030-DET

## Story
As a maintainer, I want strict mode to fail closed, so unsupported execution does not masquerade as deterministic.

## Acceptance Criteria
- Forbidden or unclassified operations fail before side effects execute.
- Error messages distinguish deterministic violations from generic runtime failures.
- Tests cover allowed and denied strict-mode paths.

## Tasks (0.5-2 days)
- E-26030-DET-S2-T1 (1.0d): Define strict-mode error types and fail-closed behavior for unsupported operations.
- E-26030-DET-S2-T2 (1.0d): Add tests for strict-pass, strict-deny, and default-mode compatibility.
