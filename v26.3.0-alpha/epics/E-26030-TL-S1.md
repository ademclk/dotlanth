# Story E-26030-TL-S1: `dot validate-replay <run>` command

## Epic

## Story
As an operator, I want a validator command, so I can check whether a replay matches the original deterministic proof surface.

## Acceptance Criteria
- `dot validate-replay <run>` loads original run artifacts, replays, and compares proof material.
- Output reports `valid`, `invalid`, or `unsupported`.
- Exit codes are stable and non-zero for invalid/tooling failure states.

- E-26030-TL-S1-T2 (1.0d): Add tests for valid, invalid, and unsupported validator outcomes.
