# Story E-26030-SEC-S2: deterministic audit events and reporting

## Epic
- E-26030-SEC

## Story
As an operator, I want deterministic violations logged as audit events, so I can explain why a run was denied or downgraded.

## Acceptance Criteria
- Violations emit structured audit events.
- Audit events include run id, opcode/category, and capability context where available.
- Failed strict runs still expose violation summaries in artifacts.

## Tasks (0.5-2 days)
- E-26030-SEC-S2-T1 (1.0d): Define and emit structured audit events for deterministic denials and controlled side-effect use.
- E-26030-SEC-S2-T2 (0.5d): Add artifact/report integration tests for violation summaries on failed runs.
