# RFC-26032: Deterministic Hooks and Audit Events

## Status
Draft (target for v26.3.0-alpha)

## Summary
Defines how deterministic-sensitive host surfaces are gated in v26.3 and how audit events are emitted for allowed or denied operations.

## Scope
- Time access
- Randomness access
- Network access
- Deterministic audit events and first-pass budget counters

## Hook Contract
- Deterministic-sensitive host access MUST flow through policy-aware hooks.
- In strict mode, hooks MUST consult determinism mode and opcode classification before host execution.
- Capability enforcement and determinism enforcement MUST both apply when relevant.

## Audit Event Schema (normative, minimal)
- `run_id` (string)
- `seq` (integer or trace-linked identifier)
- `event_kind` (string)
- `opcode` (string)
- `determinism_category` (string)
- `capability` (string or null)
- `decision` (string: `allowed` or `denied`)
- `reason` (string)

## Budget Counters (v26.3 informational)
- `gated_time_calls`
- `gated_random_calls`
- `gated_network_calls`
- `denied_non_deterministic_ops`
- `allowed_controlled_side_effect_ops`

These counters are informational in v26.3 and MUST NOT be interpreted as hard quotas.
